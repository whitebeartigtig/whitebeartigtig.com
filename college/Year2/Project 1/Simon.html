Setup

Here is Unity Hub. I have already signed in wiht my account. This is where a list of projects are and where you can manage your Unity installs in the case that you have more than one version of Unity installled on your machine.

On a new user account you may need to locate the existing install location, but since I use my personal machine, it was already here.

To create a new project, you can click on the big blue "New project" button. This window will pop up, you can select your editor version here (in this case 2022.3.44f1), select "3D (Built-In Render Pipeline), name the project and then press on "Create project".

The next step is importing any assets we may use. In this case we have some sample assets. You can simply import them by dragging the folder into the Project window.

I will also take the time to mention the Hierarchy window, this is where all of the objects in your current scene are located. By default there will be the "Main Camera" and a "Directional Light".

Now that we've got the project created and assets imported, we can create a new Terrain by going to GameObject > Terrain.

Right now it is just flat, so to change that you go under the Inspector widget and select Raise or Lower Terrain.

With the terrain object selected, you can now draw the terrain in, Holding down the left click button for longer will make it taller. To make it smaller, you can hold shift and do the same thing.

Smooth Height does just about what you would expect, it smoothes out the terrain. It is most effective on the jagged edges that sometimes arrise.

Set height allows you to set a maximum height, and it will never go above the height entered.

Next up is adding textures. Go to paint textures, Create Layer and then

Rain Effect

This can be added to the scene and it will add a rain particle effect. This is simply an asset that another person has made.

The rain effect can be controlled by opening un the heirarchy and modifying the properties in Inspector.

You can add normal and mask maps to a texture which makes them look less flat.

Skybox

To create a skybox, we need to make a new photoshop file with the dimensions of 8192x6144 at 100 pixels per centimeter. Then we can paste the image in and crop it so it fits perfectly.

We can then create a rectangle with the dimensions of 2048x2048, and paste that 6 times, and position them.

After that we need to offset the image itself to make a seamless image. Go to Filter > Other > Offset.

Post Processing

Post Processing works about the same way you'd expect it to work in photo editing software. To use this you will need to install Post Processing from the package manager, and then apply a Post-process Layer and Post-process Volume. In the Post-process Volume, you can add various effects such as Bloom, Colour Grading, Depth of Field, etc. These can all contribute to making the game look much better.

Mini Game 1

Planning

I planned out a mini game to make within an hour, it was mostly successful, but animating all of those platforms took too long, so I didn't complete it. Given an extra 15-20 mins I believe I could complete the rest of this game, as all that needs to be done now is add in the checkpoints and play test it.

Animation Potential

Right now, the animations simply move side to side, and the player just needs to jump onto them. They make it more difficult, as timing is now a factor in the gameplay. What I could do is use more different movements such as circular movements and going up and down, much like Fall Guys and how it does it. This would give the game more variety in the gameplay and make it more fun for the player.

The game itself

Here is a screen recording of me playing the game.

https://youtu.be/rj16o_VaHP8

The game is made up of platforms, some moving, some still. The larger platforms are the checkpoints, if you fall off, you will be sent back to the last checkpoint. The game play works mostly as intended, although it is possible to cheat and go to a future stage before completing the previous ones, but it is a much harder jump than what is in the gameplay so I will just embrace that rather than fixing it.

Reflection

Whilst this game was a good project to work on for developing skills, it was a little over ambitious for the allotted time (1 hour). Next time I would make something that is smaller, and maybe has a little more variety in its animations. If I was to expand on this project in the future, what I would do is add more variety of different animations, and change the textures of the platforms, and perhaps make the platforms at the bottom which kill the player invisible. I would go for something more colourful like fall guys, I feel that those bright colours would go great with the chaotic nature of what I have created.

Peer Feedback

The first bit of feedback reporting a potential bug, I can't seem to reproduce, but as soon as I see the issue I will try and fix it. The second piece of feedback I completely agree with, it is difficult, but can be made easier by using different colours/textures for the platforms. This is a change I will definitely make going forward as it is something that has worked in a game before (Fall Guys), so if done right it would look good and make the game a much better experience.

Playing other games has helped me realise the importance of texturing the platforms, but it is always possible to over do it.

The Island

Research

Plan

For this project, I will create a volcanic island with a small village located in the caldera. It will have many trees, and the dock will be located on the beach, which is the only beach on the island, with a walk way up to the top.
There will also be caves on this island, along with hot springs.

Proposal Statement

My game is a short story-based adventure game with the goal of finding the source of the severed hands that have been terrorizing the residents of the village for generations.
The player will start on the beach at the port with the boat leaving them behind. The player wins when they destroy the source of the hands deep inside the caves. There will also be a pop up on screen congratulating the player on succeeding, which will give them the option to go back to the title screen or to continue exploring the island.

The island is volcanic, and the village named Aurax is located inside the caldera. The structures in the village are traditional Japanese style. It will also have many trees, and the ground will be grass. There will also be a hot spring on the island.

In addition to that, there will also be the caves which are where the hands come from, the source will be deep within. The caves will be made up of volcanic rock and be dark. The player will have to hold a light source, such as a torch, to explore down there. They can get that from the village.

The atmosphere of the game will change depending on where you are on the island. If you are in the hot springs, it will be a calm and relaxing, with just the sounds of the bubbling water. On the beach, the sound of the waves crashing up against the rocks, and seagulls can be heard, and in the main village/forest area, you can hear birds tweeting, and the occasional scuttling of the hands. Finally, in the cave, you can hear the echoes of your own footsteps and the scuttling of hands occasionally.

The main threat in the game is the hands, they will attack the player, and if they succeed in that, then the player will die, and they will be sent back to the beginning of the game.
The research I am using for this game is my secondary research on islands, and traditional Japanese architecture. Along with my primary research which included taking pictures for some textures in the game.

The assets that are important to my game, is a player controller, trees, rocks, the houses, and the severed hands. These are the main things I expect to need to create this game.
Lastly, the visual language of this game will use textures to create a semi-realistic environment in which a fantasy creature, the severed hands, will exist in. The lighting and fog will create a strong atmosphere, which will change depending on the area of the map.
